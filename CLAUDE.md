# festival_recap — Claude Project Context

## What this is
Self-hosted festival recap video generator. Node 20/Express + MariaDB,
Dockerised, running standalone on the **Raspberry Pi** (gatekeeper,
192.168.178.37) behind Caddy. URL: `https://festival_recap.homeserver.fritz.box/`

**Independent project — nothing to do with Evestival or job_search**, beyond
sharing the Pi's `mariadb` container and `web` Docker network (own isolated
database, own user, no cross-database access).

## Deploy workflow
1. `bash deploy/setup-pi.sh` — one-time bootstrap (clone, secrets, data dir)
2. Fill `secrets/db_password.txt` manually on Pi
3. Create DB + user in MariaDB (see `scripts/sql/000_create_db_user.sql`)
4. Add tracks under `music/`, run `node scripts/seed-music.js`
5. Append `caddy/festival_recap.caddy` to Pi Caddyfile → reload Caddy
6. `bash deploy/update-pi.sh` — build + start
7. `bash deploy/update-pi.sh --install-cron` — optional daily retention cleanup

## Stack
- Node 20 / Express, mysql2, multer (uploads), fluent-ffmpeg, sharp, pino logging
- Media scoring: `sharp` (sharpness/brightness/contrast) + ffprobe
  `blurdetect`/`signalstats` (video) + OpenCV Haar cascade via Python
  subprocess (`scripts/face_count.py`) for face/crowd count — all CPU-only,
  no GPU, see `docs/ARCHITECTURE.md` §5 for the cloud-API upgrade path
- No Redis/BullMQ — render worker is an in-process DB-polling loop
  (`src/queue/worker.js`), `RENDER_CONCURRENCY=1` by default
- Music: local curated library (`music/library.json` + audio files), no
  external API call at render time. Adding tracks: the "Music" tab lets you
  paste a `pixabay.com/music/...` URL — server scrapes the page's JSON-LD
  `AudioObject` block (title/artist/duration/direct mp3 URL), downloads the
  file into `music/`, and upserts the DB row (`src/services/musicImport.js`,
  `src/routes/music.js` preview/import endpoints). BPM is never published
  anywhere (Pixabay or otherwise) — always a user-editable genre-typical
  guess, confirmed before saving, never presented as measured. Only Pixabay
  URLs are understood; other sources use the manual `library.json` +
  `scripts/seed-music.js` path (see `music/README.md`).

## Known gaps (v1 baseline, not yet validated against real footage)
- No auth on the API (LAN-only via Caddy `tls internal` — add a token/JWT
  gate before any public exposure)
- Render time on real Pi hardware unmeasured — get a real number before
  promising users a turnaround
- Scoring weights in `src/services/mediaAnalysis.js` are initial estimates,
  not tuned against real festival photos/clips
- `music/library.json` ships with a placeholder entry only — use the Music
  tab (Pixabay URL import) or the manual path to add real tracks before this
  can render anything
- Pixabay scraping (`src/services/musicImport.js`) is untested from the Pi's
  actual network — Pixabay may apply bot protection that blocks server-side
  requests even with browser-like headers; if `GET /api/music/preview`
  errors out, fall back to the manual `library.json` path
